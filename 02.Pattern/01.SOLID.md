# Brief
- **What is this:** A set of five object-oriented design principles — Single Responsibility, Open/Closed, Liskov Substitution, Interface Segregation, and Dependency Inversion — that guide writing code that is easy to change, extend, and test.
- **Ultimate Goal:** Build systems where adding a feature or fixing a bug touches only the code it should, without cascading breakages — so the codebase stays healthy as it grows.

---

# Details
## S — Single Responsibility
> *"A class should have only one reason to change."*

- Explain: Each class/module handles **one concern only**
- Goals: Splitting by responsibility makes code easier to test and maintain
- Example: DialogueVolume and QuickControl and AudioManagerHelper --> `after that we applied dual headphone`

---

## O — Open/Closed
> *"Software entities should be open for extension, but closed for modification."*

- Explain: Add new behavior by **adding block**, not break or mess the structure
- Goals: Use abtraction and polymorphism to extend the code
- Example: FOD service - I implemented state pattern. 

---

## L — Liskov Substitution
> *"Objects of a subclass should be replaceable with objects of the superclass without breaking correctness."*

- Explain: Subclass should not break or change correctness of parents

## I — Interface Segregation
> *"Clients should not be forced to depend on methods they do not use."*

- Split large interfaces into **smaller, role-specific** ones
- A class implements only what it needs
- Example: TODO  (about a refactor and it helps later)

---

## D — Dependency Inversion
> *"High-level modules should not depend on low-level modules. Both should depend on abstractions."*

- Code to **interfaces/abstractions**, inject concrete implementations
- Makes swapping implementations (real DB vs mock) trivial
- Example: AIDL files
