# CI/CD, Quality, and Tooling Q&A

## Covered Areas
- CI/CD in automotive
- Developer tooling
- Git workflow
- Code quality
- ASPICE

## Questions

### Q41. What does CI/CD mean in embedded or automotive software?
It means automating build, validation, packaging, and delivery as much as the platform allows.

### Q42. What is the value of developer environment automation?
It reduces onboarding time, setup errors, and productivity loss across teams.

### Q43. How would you answer a question about Git in interviews?
Focus on branching, code review, collaboration, and using history safely for debugging.

### Q44. What is your approach to code quality in multi-team projects?
Use strong boundaries, reviews, logging, automation, and clear interface ownership.

### Q45. What is ASPICE, and why does it matter?
ASPICE is an automotive process framework that supports traceability, reviewability, and engineering quality.
