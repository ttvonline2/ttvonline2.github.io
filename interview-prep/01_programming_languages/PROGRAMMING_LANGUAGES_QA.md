# Programming Languages Q&A

## Covered Areas
- Kotlin
- Java
- C++
- Python

## Questions

### Q01. Why is Kotlin a strong choice for Android development?
Kotlin reduces boilerplate, improves null safety, supports coroutines, and integrates well with Android APIs.

### Q02. What is the difference between `val` and `var` in Kotlin?
`val` is read-only after initialization, while `var` is mutable.

### Q03. How does Kotlin null safety help in production code?
It forces explicit handling of nullable values and prevents many runtime null crashes.

### Q04. When would you use coroutines in Android?
Use coroutines for asynchronous work such as I/O, background processing, and lifecycle-aware tasks.

### Q05. How would you explain Kotlin vs Java in an interview?
Java is stable and widely used, while Kotlin improves safety and productivity for modern Android development.

### Q06. Why is Java still relevant in Android projects?
Many frameworks, libraries, and legacy enterprise codebases still depend on Java.

### Q07. What is the difference between an interface and an abstract class in Java?
An interface defines a contract, while an abstract class can also provide shared state and partial implementation.

### Q08. What are common Java performance concerns on Android?
Heavy allocations, reflection, bad threading choices, and too much work on the main thread.

### Q09. Why is C++ useful in automotive or embedded systems?
It provides strong control over memory, performance, and low-level system behavior.

### Q10. What are the tradeoffs between C++ and Kotlin or Java?
C++ gives more control and performance, while Kotlin and Java improve productivity and safety for application logic.

### Q11. What are common causes of instability in C++ systems?
Memory leaks, dangling pointers, race conditions, and unclear ownership.

### Q12. How have you used Python in engineering work?
Mainly for scripting, automation, environment setup, and developer productivity tools.
