# Soong Build System

## 1. Tổng quan

**Soong** là hệ thống build hiện đại của Android, được Google phát triển để thay thế hệ thống build cũ dựa trên **GNU Make**. Soong được viết bằng **Go** và sử dụng file cấu hình có cú pháp **Blueprint** (`.bp`).

### Lịch sử phát triển

| Phiên bản Android | Hệ thống build |
|---|---|
| Android 1.x – 6.x | GNU Make (`Android.mk`) |
| Android 7.x – 9.x | Hybrid (Make + Soong) |
| Android 10+ | Soong là chủ đạo (`Android.bp`) |
| Android 12+ | Make gần như bị loại bỏ hoàn toàn |

### Tại sao cần Soong?

- **GNU Make** chậm, khó bảo trì và dễ gây lỗi khi project phức tạp.
- Soong hỗ trợ **incremental build** và **parallel build** hiệu quả hơn.
- Cú pháp Blueprint (`.bp`) rõ ràng, dễ đọc hơn Makefile.
- Tích hợp tốt với **Ninja build system** để thực thi các bước build.

## 2. Kiến trúc tổng thể

```
Developer / CI
      │
      ▼
┌─────────────┐
│  Android.bp │  ← File cấu hình (Blueprint syntax)
└──────┬──────┘
       │  Soong (Go)
       ▼
┌─────────────┐
│  build.ninja│  ← File build được sinh ra
└──────┬──────┘
       │  Ninja
       ▼
┌─────────────┐
│  Output     │  ← .apk, .so, .jar, .img, ...
└─────────────┘
```

### Các thành phần chính

| Thành phần | Vai trò |
|---|---|
| `Android.bp` | File khai báo module, viết bằng Blueprint syntax |
| `soong_build` | Binary Go chính, đọc `.bp` và sinh `build.ninja` |
| `Blueprint` | Ngôn ngữ cấu hình + parser được Soong sử dụng |
| `Ninja` | Thực thi các bước build từ `build.ninja` |
| `Kati` | Công cụ chuyển đổi `Android.mk` → Ninja (dùng để tương thích ngược) |
| `Bazel` (Android 12+) | Tích hợp thêm để build một số module quan trọng |

## 3. Cú pháp Blueprint (Android.bp)

File `Android.bp` sử dụng cú pháp **Blueprint** — một ngôn ngữ khai báo đơn giản, không phải ngôn ngữ lập trình đầy đủ.

### Cấu trúc cơ bản

```bp
module_type {
    name: "module_name",
    property1: "value1",
    property2: ["value2a", "value2b"],
}
```

### Ví dụ thực tế

```bp
// Thư viện C++ tĩnh
cc_library_static {
    name: "libmyutil",
    srcs: ["src/util.cpp"],
    export_include_dirs: ["include"],
}

// Binary thực thi C++
cc_binary {
    name: "myapp",
    srcs: ["main.cpp"],
    static_libs: ["libmyutil"],
}

// Thư viện Java
java_library {
    name: "myjavalib",
    srcs: ["src/**/*.java"],
    sdk_version: "current",
}

// APK Android App
android_app {
    name: "MyApp",
    srcs: ["src/**/*.java"],
    resource_dirs: ["res"],
    manifest: "AndroidManifest.xml",
    sdk_version: "current",
}
```

## 4. Các module type quan trọng

### 4.1 Module C/C++

| Module Type | Mô tả |
|---|---|
| `cc_binary` | Binary thực thi (native executable) |
| `cc_library_shared` | Thư viện chia sẻ (`.so`) |
| `cc_library_static` | Thư viện tĩnh (`.a`) |
| `cc_library` | Vừa shared vừa static |
| `cc_test` | Unit test C++ (chạy trên device) |
| `cc_defaults` | Template mặc định cho cc_* modules |

```bp
cc_library_shared {
    name: "libexample",
    srcs: [
        "foo.cpp",
        "bar.cpp",
    ],
    shared_libs: [
        "liblog",
        "libcutils",
    ],
    cflags: ["-Wall", "-Werror"],
    include_dirs: ["include"],
}
```

### 4.2 Module Java / Kotlin

| Module Type | Mô tả |
|---|---|
| `java_library` | Thư viện Java |
| `java_binary` | Binary Java (JAR thực thi) |
| `android_library` | Thư viện Android (AAR) |
| `android_app` | Ứng dụng Android (APK) |
| `android_test` | Test APK chạy trên device |
| `java_defaults` | Template mặc định cho java_* modules |

```bp
android_app {
    name: "MySystemApp",
    srcs: ["src/**/*.kt"],
    resource_dirs: ["res"],
    manifest: "AndroidManifest.xml",
    platform_apis: true,          // Dùng API nội bộ Android
    system_ext_specific: true,    // Cài vào /system_ext
    certificate: "platform",      // Ký bằng platform key
    privileged: true,             // Cài vào /system/priv-app
}
```

### 4.3 Module AIDL / HIDL / AIDL-cpp

```bp
// AIDL Interface
aidl_interface {
    name: "android.hardware.example",
    vendor_available: true,
    srcs: ["android/hardware/example/*.aidl"],
    stability: "vintf",
    versions: ["1"],
}
```

### 4.4 Prebuilt modules

```bp
// File APK prebuilt
android_app_import {
    name: "ThirdPartyApp",
    apk: "prebuilt/ThirdPartyApp.apk",
    presigned: true,
    dex_preopt: {
        enabled: false,
    },
}

// Thư viện .so prebuilt
cc_prebuilt_library_shared {
    name: "libthirdparty",
    srcs: ["libs/libthirdparty.so"],
    strip: {
        none: true,
    },
}
```

## 5. Các thuộc tính cấu hình quan trọng

### 5.1 `defaults` — Tái sử dụng cấu hình

`defaults` cho phép khai báo một tập cấu hình dùng chung và áp dụng cho nhiều module.

```bp
cc_defaults {
    name: "my_cc_defaults",
    cflags: [
        "-Wall",
        "-Wextra",
        "-DDEBUG_BUILD",
    ],
    shared_libs: ["liblog"],
}

cc_binary {
    name: "app1",
    defaults: ["my_cc_defaults"],
    srcs: ["app1.cpp"],
}

cc_binary {
    name: "app2",
    defaults: ["my_cc_defaults"],
    srcs: ["app2.cpp"],
}
```

### 5.2 `visibility` — Kiểm soát phạm vi truy cập

`visibility` kiểm soát module nào được phép phụ thuộc vào module này.

```bp
cc_library {
    name: "libinternal",
    visibility: [
        "//packages/apps/MyApp",      // Chỉ module trong thư mục này
        "//vendor/mycompany:__pkg__", // Chỉ package đó
        "//visibility:public",        // Mở cho tất cả
        "//visibility:private",       // Chỉ cùng package
    ],
    srcs: ["lib.cpp"],
}
```

### 5.3 `apex_available` — Tích hợp với APEX

Khai báo module có thể được đóng gói trong **Android APEX** container.

```bp
cc_library {
    name: "libapexmodule",
    srcs: ["lib.cpp"],
    apex_available: [
        "com.android.myapex",
        "//apex_available:platform", // Cũng có thể dùng ngoài APEX
    ],
}
```

### 5.4 `target` — Cấu hình theo platform

`target` cho phép cấu hình khác nhau tùy theo platform (android, linux, host, v.v.)

```bp
cc_binary {
    name: "mytool",
    srcs: ["main.cpp"],
    target: {
        android: {
            shared_libs: ["libandroid"],
            cflags: ["-DANDROID"],
        },
        linux: {
            cflags: ["-DLINUX"],
        },
        host: {
            static_libs: ["libhost_utils"],
        },
        android_arm64: {
            cflags: ["-DARM64"],
        },
    },
}
```

### 5.5 `arch` — Cấu hình theo kiến trúc CPU

```bp
cc_library {
    name: "libarch_specific",
    srcs: ["generic.cpp"],
    arch: {
        arm: {
            srcs: ["arm_impl.S"],
        },
        arm64: {
            srcs: ["arm64_impl.S"],
        },
        x86: {
            srcs: ["x86_impl.cpp"],
        },
        x86_64: {
            srcs: ["x86_64_impl.cpp"],
        },
    },
}
```

### 5.6 `product_variables` — Cấu hình theo biến sản phẩm

Cho phép thay đổi cấu hình dựa trên các biến build `BoardConfig.mk` / `device.mk`.

```bp
cc_library {
    name: "libfeature",
    srcs: ["feature.cpp"],
    product_variables: {
        debuggable: {
            cflags: ["-DENABLE_DEBUG_LOGS"],
        },
        treble_linker_namespaces: {
            cflags: ["-DTREBLE_LINKER"],
        },
    },
}
```

### 5.7 `stl` — Chọn C++ Standard Library

```bp
cc_library {
    name: "libcppexample",
    srcs: ["example.cpp"],
    stl: "libc++",        // libc++ (mặc định cho Android)
    // stl: "libc++_static"  // Static link libc++
    // stl: "none"           // Không dùng STL
    // stl: "system"         // STL của hệ thống
}
```

### 5.8 `sdk_version` và `platform_apis`

Dành cho module Java/Kotlin, kiểm soát API set được phép sử dụng.

```bp
// App thông thường - chỉ dùng public API
android_app {
    name: "NormalApp",
    sdk_version: "current",       // API public mới nhất
    // sdk_version: "30"          // API level cụ thể
    // sdk_version: "system_current" // System API
    // sdk_version: "test_current"   // Test API
}

// System app - dùng internal API
android_app {
    name: "SystemApp",
    platform_apis: true,          // Mở toàn bộ platform API (không khuyến khích)
}
```

### 5.9 `certificate` — Ký APK

```bp
android_app {
    name: "MyApp",
    certificate: "platform",   // Ký bằng platform key
    // certificate: "shared"   // Shared key
    // certificate: "media"    // Media key
    // certificate: "testkey"  // Test key (default)
    // certificate: ":mykey"   // Custom key trong Android.bp
}
```

### 5.10 `init_rc` — Khai báo Service

```bp
cc_binary {
    name: "mydaemon",
    srcs: ["daemon.cpp"],
    init_rc: ["mydaemon.rc"],  // Tự động cài file .rc vào /system/etc/init/
}
```

## 6. Soong Namespace

Namespace tránh xung đột tên module khi có nhiều vendor/partner cùng build.

```bp
// Android.bp tại root của vendor module
soong_namespace {
    imports: [
        "vendor/google/libs",   // Import namespace khác
    ],
}
```

Tham chiếu module từ namespace khác:

```bp
cc_binary {
    name: "myvendorapp",
    static_libs: [
        "//vendor/google/libs:libshared",  // Fully-qualified name
    ],
}
```

## 7. Build Variables & Configuration Files

### 7.1 Luồng cấu hình

```
BoardConfig.mk          device/<vendor>/<product>/
    └─► device.mk       ← Product configuration
           └─► Android.bp modules sử dụng product_variables
```

### 7.2 Các file cần biết

| File | Vị trí | Vai trò |
|---|---|---|
| `Android.bp` | Mọi nơi trong source tree | Khai báo modules |
| `BoardConfig.mk` | `device/<vendor>/<device>/` | Cấu hình hardware |
| `device.mk` | `device/<vendor>/<device>/` | Danh sách packages |
| `build/soong/Android.bp` | AOSP root | Khai báo toolchain |
| `build/make/core/` | AOSP root | Logic build cốt lõi |

## 8. Lệnh build thường dùng

```bash
# Thiết lập môi trường build
source build/envsetup.sh

# Chọn target
lunch aosp_arm64-eng

# Build toàn bộ hệ thống
m

# Build một module cụ thể
m ModuleName

# Build và chạy test
m ModuleName && adb sync

# Kiểm tra dependency graph
soong_ui --make-mode --skip-make queryCmd -- ModuleName

# Xem các module được khai báo
m nothing 2>&1 | grep "ModuleName"
```

## 9. So sánh Android.mk vs Android.bp

| Tiêu chí | Android.mk | Android.bp |
|---|---|---|
| Ngôn ngữ | GNU Make | Blueprint (JSON-like) |
| Tốc độ parse | Chậm | Nhanh |
| Xử lý logic | Có (if/else, loops) | Không (khai báo thuần túy) |
| Tích hợp Ninja | Gián tiếp (qua Kati) | Trực tiếp |
| Đọc hiểu | Khó | Dễ |
| Hỗ trợ tooling | Hạn chế | Tốt (IDE, lint, v.v.) |
| Trạng thái | Deprecated | Chính thức |
