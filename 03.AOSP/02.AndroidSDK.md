# Android SDK trong AOSP

## 1. SDK là gì?

**SDK (Software Development Kit)** là tập hợp các công cụ, thư viện, API, và tài liệu cho phép lập trình viên xây dựng ứng dụng trên nền tảng Android.

### 1.1 Hai loại SDK trong ngữ cảnh AOSP

```
┌────────────────────────────────────────────────────────┐
│                   Android SDK                          │
├───────────────────────┬────────────────────────────────┤
│  Public SDK           │  Platform SDK (Internal)       │
│  (developers.android) │  (AOSP source tree)            │
├───────────────────────┼────────────────────────────────┤
│ Dành cho app          │ Dành cho system/vendor app     │
│ developer bên ngoài   │ trong AOSP                     │
│                       │                                │
│ Được ổn định giữa các │ Có thể thay đổi theo từng     │
│ version Android       │ AOSP release                   │
│                       │                                │
│ android.jar (public)  │ framework.jar, services.jar    │
│ sdk_version: "30"     │ platform_apis: true            │
└───────────────────────┴────────────────────────────────┘
```

### 1.2 SDK gồm những gì?

| Thành phần | Vị trí trong AOSP | Mô tả |
|---|---|---|
| **Framework API** | `frameworks/base/` | API Java/Kotlin cho app |
| **android.jar** | `out/target/common/obj/JAVA_LIBRARIES/android_stubs_current_intermediates/` | Stub JAR để compile app |
| **AIDL interfaces** | `frameworks/base/core/java/` | IPC interface |
| **NDK headers** | `prebuilts/ndk/` | C/C++ headers cho native |
| **System services** | `frameworks/base/services/` | ActivityManager, PackageManager, v.v. |
| **Build tools** | `build/soong/`, `build/make/` | Soong, Ninja, dx, aapt2 |
| **Platform tools** | `packages/modules/adb/` | adb, fastboot |

### 1.3 Luồng từ Framework đến App

```
AOSP Source (frameworks/base/)
         │
         ▼
   Compile → framework.jar  ← đây là Platform SDK (đầy đủ)
         │
         ▼
   Generate stubs → android.jar  ← đây là Public SDK (che dấu @hide)
         │
         ▼
   App dùng android.jar để compile
         │
         ▼
   Runtime: App gọi vào framework.jar thực sự trên device
```

## 2. Tại sao cần chỉnh sửa SDK?

Dưới đây là các tình huống thực tế buộc phải chỉnh sửa SDK:

### 2.1 Bổ sung API mới cho System App

Khi bạn phát triển một **System App** cần gọi API nội bộ (`@hide`) không có trong Public SDK.

```
Vấn đề:
App cần gọi ActivityManager.forceStopPackage()
→ Method này bị @hide trong Public SDK
→ Compile error nếu dùng sdk_version: "current"

Giải pháp:
→ Dùng platform_apis: true trong Android.bp
→ Hoặc expose API đó qua System API (@SystemApi)
```

### 2.2 Tùy chỉnh hành vi Framework cho sản phẩm OEM

OEM (Samsung, Xiaomi, v.v.) thường:
- Thêm tính năng riêng vào framework
- Sửa đổi API hiện có
- Bổ sung thư viện đặc thù phần cứng

### 2.3 Thêm System Service mới

Khi tích hợp phần cứng đặc biệt (sensor, codec, v.v.) cần:
- Viết System Service mới
- Expose API lên Framework layer
- App gọi qua Manager class

### 2.4 Fix lỗi hoặc patch bảo mật

Google Security Bulletin hàng tháng yêu cầu:
- Patch lỗ hổng trong `frameworks/base/`
- Rebuild SDK và các phụ thuộc

### 2.5 Thay đổi API behavior (không tương thích ngược)

```
Ví dụ: Camera2 API thêm constraint mới
→ Vendor driver cũ không đáp ứng
→ Phải sửa CameraService trong framework
→ Rebuild toàn bộ camera stack
```

## 3. Vị trí code SDK quan trọng

```
frameworks/
├── base/
│   ├── core/java/android/          ← Core API (Activity, Context, Intent...)
│   ├── services/core/java/         ← System Services (AMS, PMS, WMS...)
│   ├── api/                        ← API surface definitions (*.txt)
│   │   ├── current.txt             ← Public API list
│   │   ├── system-current.txt      ← System API list
│   │   └── removed.txt             ← Removed APIs
│   ├── cmds/                       ← Command line tools
│   └── libs/                       ← Native libraries
├── av/                             ← Media framework (MediaPlayer, Camera)
├── native/                         ← Native services (SurfaceFlinger, AudioFlinger)
└── opt/
    └── telephony/                  ← Telephony stack
```

## 4. Cách chỉnh sửa SDK

### 4.1 Thêm method mới vào Framework API

**Bước 1:** Mở file Java trong `frameworks/base/core/java/android/`

```java
// frameworks/base/core/java/android/app/ActivityManager.java

public class ActivityManager {

    // ... existing code ...

    /**
     * Lấy tên tiến trình hiện tại.
     * @hide  ← ẩn khỏi Public SDK, chỉ dùng nội bộ
     */
    public String getCurrentProcessName() {
        return Application.getProcessName();
    }

    /**
     * @SystemApi  ← expose lên System SDK
     * @hide
     */
    public void customOemFunction(String param) {
        // implementation
    }
}
```

**Bước 2:** Nếu muốn expose là Public API, **xóa `@hide`** và update API list:

```bash
cd $ANDROID_BUILD_TOP
m update-api   # Cập nhật frameworks/base/api/current.txt
```

**Bước 3:** Nếu chỉ expose System API, thêm `@SystemApi`:

```java
/**
 * @SystemApi
 * @hide
 */
public void mySystemApi() { ... }

# Sau đó chạy:
m update-api
```

### 4.2 Thêm System Service mới

**Bước 1:** Tạo AIDL interface

```
// frameworks/base/core/java/android/app/IMyService.aidl
package android.app;

/** @hide */
interface IMyService {
    String getDeviceInfo();
    void setFeatureEnabled(boolean enabled);
}
```

**Bước 2:** Implement Service

```java
// frameworks/base/services/core/java/com/android/server/MyService.java
public class MyService extends IMyService.Stub {

    @Override
    public String getDeviceInfo() {
        return Build.MODEL + " / " + Build.VERSION.RELEASE;
    }

    @Override
    public void setFeatureEnabled(boolean enabled) {
        // implementation
    }
}
```

**Bước 3:** Đăng ký Service trong `SystemServer.java`

```java
// frameworks/base/services/java/com/android/server/SystemServer.java

private void startOtherServices() {
    // ... existing services ...

    traceBeginAndSlog("StartMyService");
    ServiceManager.addService("my_service", new MyService(context));
    traceEnd();
}
```

**Bước 4:** Tạo Manager class cho app

```java
// frameworks/base/core/java/android/app/MyServiceManager.java
public class MyServiceManager {
    private static IMyService sService;

    public static String getDeviceInfo() {
        return getService().getDeviceInfo();
    }

    private static IMyService getService() {
        if (sService == null) {
            sService = IMyService.Stub.asInterface(
                ServiceManager.getService("my_service"));
        }
        return sService;
    }
}
```

**Bước 5:** Đăng ký trong `Context.java` (nếu muốn dùng `getSystemService()`)

```java
// frameworks/base/core/java/android/app/SystemServiceRegistry.java
registerService(Context.MY_SERVICE, MyServiceManager.class,
    new CachedServiceFetcher<MyServiceManager>() {
        @Override
        public MyServiceManager createService(ContextImpl ctx) {
            return new MyServiceManager();
        }
    });
```

### 4.3 Patch lỗi trong Framework

```bash
# Ví dụ: Fix bug trong PackageManagerService
vi frameworks/base/services/core/java/com/android/server/pm/PackageManagerService.java

# Kiểm tra compile trước khi build full
m framework-minus-apex
```

## 5. Cách build lại SDK

### 5.1 Thiết lập môi trường

```bash
# Bắt buộc thực hiện mỗi khi mở terminal mới
cd $ANDROID_BUILD_TOP   # root của AOSP source
source build/envsetup.sh
lunch <target>          # Ví dụ: lunch aosp_arm64-eng
```

### 5.2 Build theo phạm vi thay đổi

#### Build chỉ Framework Java

```bash
# Build framework.jar (nhanh nhất, ~5-15 phút)
m framework
```

#### Build Framework + Services

```bash
# Khi sửa System Service
m framework services
```

#### Build lại android.jar (Public SDK stub)

```bash
# Khi thêm/xóa Public API
m android_stubs_current
```

#### Build toàn bộ SDK

```bash
# Khi làm SDK release đầy đủ
m sdk
```

#### Build module cụ thể

```bash
# Tên module theo Android.bp
m framework-minus-apex      # framework không có APEX modules
m services                  # System services
m core-oj                   # Core Java (java.lang, java.io, ...)
```

### 5.3 Kiểm tra API compatibility

```bash
# Bắt buộc trước khi submit code liên quan API
m checkapi

# Nếu có API mới, cần update API definition trước
m update-api && m checkapi
```

### 5.4 Flash lên thiết bị

```bash
# Sau khi build xong
adb root
adb remount   # Mount /system read-write

# Sync chỉ framework files (không cần flash toàn bộ image)
adb sync system

# Hoặc push thủ công
adb push out/target/product/<device>/system/framework/framework.jar /system/framework/

# Khởi động lại
adb shell stop
adb shell start
# Hoặc
adb reboot
```

### 5.5 Build image đầy đủ (khi release)

```bash
# Build system image
m systemimage

# Build tất cả images
m

# Flash toàn bộ
fastboot flashall -w
```

## 6. Kiểm tra và Debug

### 6.1 Kiểm tra API changes

```bash
# So sánh API hiện tại vs baseline
diff frameworks/base/api/current.txt <(m -n api-stubs-docs-non-updatable 2>/dev/null)

# Xem API bị xóa
cat frameworks/base/api/removed.txt
```

### 6.2 Xem log khi build

```bash
# Build với verbose output
m framework 2>&1 | tee build_log.txt

# Tìm lỗi compile
m framework 2>&1 | grep -E "error:|warning:"
```

### 6.3 Dùng adb để debug System Service

```bash
# Dump service info
adb shell dumpsys my_service

# Gọi service qua shell
adb shell service call my_service 1          # Gọi method đầu tiên

# Xem log của framework
adb logcat -s ActivityManager:D PackageManager:D
```

### 6.4 Kiểm tra method @hide có accessible không

```bash
# Trên device
adb shell cmd package list packages

# Dùng reflection để gọi @hide method (chỉ test)
Method m = ActivityManager.class.getDeclaredMethod("forceStopPackage", String.class);
m.setAccessible(true);
m.invoke(activityManager, "com.example.app");
```

## 7. Quy trình làm việc tổng thể

```
1. Xác định yêu cầu
        │
        ▼
2. Tìm file cần sửa trong frameworks/base/ (hoặc frameworks/av/, v.v.)
        │
        ▼
3. Chỉnh sửa code
        │
        ▼
4. Build incremental: m <module_name>
        │
        ├── Lỗi compile? → Sửa và quay lại bước 3
        │
        ▼
5. Nếu thêm API mới: m update-api
        │
        ▼
6. Chạy: m checkapi (kiểm tra tương thích)
        │
        ▼
7. Push lên device: adb sync system + adb reboot
        │
        ▼
8. Kiểm tra hoạt động: adb logcat, adb shell dumpsys
        │
        ▼
9. Build full image để release: m systemimage
```

## 8. Các lỗi thường gặp

| Lỗi | Nguyên nhân | Cách sửa |
|---|---|---|
| `error: cannot find symbol` | Import thiếu hoặc sai class name | Kiểm tra import trong file Java |
| `API check failed` | Thêm API mới chưa update api txt | Chạy `m update-api` |
| `Removed API` error | Xóa API đang được dùng | Giữ lại API, đánh dấu `@Deprecated` |
| `@hide access` error | System app dùng `@hide` API với `sdk_version` | Đổi sang `platform_apis: true` |
| Build OOM | RAM không đủ khi build | `export JACK_SERVER_VM_ARGUMENTS="-Xmx4g"` hoặc giảm `-j` |
| `AIDL not found` | File `.aidl` chưa được khai báo trong `Android.bp` | Thêm vào `srcs` của module |
